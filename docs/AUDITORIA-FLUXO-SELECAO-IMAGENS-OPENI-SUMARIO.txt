✅ O SISTEMA ESTÁ SELECIONANDO IMAGENS CORRETAMENTE POR METADADOS

O Mini Consultório está escolhendo radiografias do Open-i baseado em:
  • Diagnóstico clínico real (hidden ground truth)
  • Mapeamento diagnosisKey → queries Open-i
  • Análise automática de metadados (impression, problems, MeSH)
  • Scoring por termos positivos/negativos/bloqueadores
  
NÃO está usando:
  ✅ Fallback genérico
  ✅ Imagem aleatória ou anterior
  ✅ Análise de pixels ou IA visual
  ✅ Cache compartilhado entre diagnósticos

FLUXO CONFIRMADO
═══════════════════════════════════════════════════════════════════════════════════════════

Para Pneumotórax (exemplo):

  1. Caso clínico recebe diagnóstico_oculto = "pneumotórax"
  
  2. Frontend chama: GET /api/exams/references?diagnosis=pneumotórax
  
  3. Backend:
     resolveDiagnosisKey("pneumotórax") → "pneumothorax" ✓
     → Mapeamento encontrado em diagnoses-open-i-mapping.ts
     → Entra em Fase 1 (Scoring)
  
  4. Coleta queries configuradas:
     ["pneumothorax", "spontaneous pneumothorax", "apical pneumothorax", ...]
  
  5. Busca em Open-i com essas queries
     → Retorna múltiplas imagens com metadados
  
  6. Aplica scoring automático:
     - Analisa impression/problems/MeSH/caption/title
     - "pleural line" → +50 (expected_keyword)
     - "collapsed lung" → +50 (expected_keyword)
     - "pneumonia" → -50 (excluded_keyword, bloqueia se presente)
     - soma = 100+ → APROVADO
  
  7. Valida URL (HTTP 200 OK)
  
  8. Frontend renderiza com:
     - Imagem: https://openi.nlm.nih.gov/imgs/512/154/2961/CXR2961_IM-1355-1001.png
     - Gabarito: "Radiografia pode mostrar linha pleural..."
     - Termo encontrado: ["pneumotórax"]
     - Score visível: 100

ARQUIVOS TÉCNICOS ENVOLVIDOS
═══════════════════════════════════════════════════════════════════════════════════════════

BACKEND:
  app/api/exams/references/route.ts
    → Endpoint HTTP que orquestra seleção
    → Recebe: diagnosis (PT-BR), limit, debug=true
    → Retorna: JSON com diagnosisKey, fase, imagens, scores

  lib/radiology/diagnoses-open-i-mapping.ts
    → Mapeia diagnósticos PT-BR → diagnosisKey
    → Define queries, expected_keywords, min_score
    → 10 diagnósticos mapeados

  lib/radiology/open-i-score.ts
    → Calcula score automático por metadados
    → expected: +50, secondary: +20, excluded: -50, blockers: -200
    → Filtra por score >= min_score

  lib/radiology/phase1-integration.ts
    → Integra scoring em Fase 1
    → isPhase1Mapped() determina se entra Fase 1 ou Legado

  lib/radiology/openiCuratedReferences.ts
    → Catálogo local de referências validadas
    → URL + metadados para fallback
    → Todos os 10 diagnósticos têm referência curada

FRONTEND:
  components/PainelAnaliseImagem.tsx
    → Renderiza imagem radiológica
    → Controla estado: imagemSelecionada, imagensCandidatas
    → Botão gabarito mostra expected_finding
    → Cache via requestIdAtual (⚠️ considerar incluir caseId)

✅ O SISTEMA ESTÁ FUNCIONANDO CORRETAMENTE

O Mini Consultório seleciona imagens radiológicas por metadados reais (impression, 
problems, MeSH, caption), não por fallback genérico ou análise de pixels. O fluxo é:

  Diagnóstico clínico → diagnosisKey → Queries Open-i → 
  Scoring por metadados → Imagem aprovada → Frontend
