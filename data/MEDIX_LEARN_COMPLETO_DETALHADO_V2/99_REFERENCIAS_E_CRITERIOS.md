# Referências e critérios de curadoria — MEDIX Learn V3

> Material educacional para estudantes de medicina e treino de raciocínio clínico/OSCE. Não substitui diretrizes locais, supervisão docente, protocolos institucionais ou atendimento médico real.

## Finalidade deste documento

Este arquivo define os critérios de qualidade usados na curadoria do **MEDIX Learn V3 Curado**.

O objetivo não é listar todas as fontes possíveis.  
O objetivo é garantir que cada trilha tenha consistência pedagógica, segurança clínica e utilidade prática para o estudante iniciante.

---

## Critério central

Toda trilha deve ensinar por mecanismo.

```text
conceito → fisiopatologia → semiologia → hipótese → decisão → reavaliação → OSCE
```

Se uma trilha apenas lista sinais e sintomas, ela ainda não está pronta.

---

# 1. Critérios pedagógicos

## 1.1. Linguagem

A linguagem deve ser:

- direta;
- médica;
- clara para iniciante;
- em segunda pessoa;
- sem frases genéricas como “o aluno deve”;
- sem excesso de parágrafos longos;
- com pontos de destaque.

Modelo correto:

```text
Você deve reconhecer hipoperfusão antes da hipotensão.
```

Evitar:

```text
O aluno deve compreender os aspectos relacionados à hipoperfusão.
```

## 1.2. Profundidade

O texto deve ser simples, mas não raso.

Cada conceito importante deve responder:

```text
o que é?
por que acontece?
como aparece?
quando fica perigoso?
como muda decisão?
```

## 1.3. Estrutura visual

Cada trilha deve conter blocos como:

```text
PONTO-CHAVE
NÃO ERRE
CUIDADO
RACIOCÍNIO
```

A leitura deve ser quebrada em blocos curtos para evitar material cansativo.

# 2. Critérios clínicos

## 2.1. Segurança primeiro

Toda trilha deve começar ou rapidamente chegar à pergunta:

```text
há sinal de gravidade?
```

Sinais de gravidade devem ser explícitos quando aplicável:

- instabilidade hemodinâmica;
- hipoxemia;
- alteração do nível de consciência;
- déficit neurológico focal;
- dor súbita intensa;
- sangramento;
- perfusão ruim;
- febre com toxemia;
- vômitos persistentes/desidratação;
- sinais respiratórios de esforço;
- sinais de choque.

## 2.2. Diagnóstico por mecanismo

A trilha deve evitar diagnóstico por memorização.

Exemplo correto:

```text
sibilo → via aérea estreitada → obstrução ao fluxo → asma/DPOC/anafilaxia/corpo estranho
```

Exemplo insuficiente:

```text
Sibilo é comum na asma.
```

## 2.3. Hipóteses perigosas

Toda trilha deve destacar diagnósticos que não podem ser perdidos.

Exemplos:

- dor torácica: SCA, dissecção, TEP, pneumotórax hipertensivo;
- febre: sepse, meningite, dengue com alarme;
- cefaleia: HSA, meningite, hipertensão intracraniana;
- dor abdominal: peritonite, obstrução, isquemia, gravidez ectópica quando aplicável;
- dispneia: asma grave, TEP, IC, pneumotórax, acidose.

# 3. Critérios dos mini-casos

Cada trilha deve conter mini-casos reais e úteis.

## 3.1. Mini-caso não pode ser template

Evitar:

```text
Paciente com apresentação típica relacionada a sepse.
```

Usar:

```text
Paciente febril, confuso, FC 128, FR 30, extremidades frias, enchimento capilar 5s e suspeita de pneumonia.
```

## 3.2. Cada mini-caso deve conter

```text
Cenário
Mecanismo principal
Achados-chave
Pergunta central
Resposta esperada
Erro comum
Ponte para OSCE
```

## 3.3. O mini-caso deve transferir raciocínio

O caso deve ensinar uma decisão, não apenas ilustrar uma doença.

Exemplo:

```text
Por que não esperar radiografia em pneumotórax hipertensivo instável?
```

---

# 4. Critérios das questões ativas

As questões ativas devem obrigar você a pensar.

Evitar perguntas genéricas:

```text
Como evitar decorar listas?
```

Preferir perguntas clínicas:

```text
SpO₂ normal exclui dispneia grave?
Por que CO₂ subindo em asma grave preocupa?
Qual dado diferencia síncope vasovagal de arrítmica?
```

Cada questão deve ter:

- pergunta direta;
- resposta esperada;
- utilidade clínica implícita ou explícita.

---

# 5. Critérios do mapa final

O mapa final deve ser útil para revisão rápida.

Evitar:

```text
DOR ABDOMINAL
├── conceito de base
├── semiologia
├── gravidade
├── mini-casos
└── ponte para OSCE
```

Usar mapas com conteúdo clínico:

```text
DOR ABDOMINAL
├── visceral
│   ├── cólica
│   └── mal localizada
├── peritoneal
│   ├── defesa
│   ├── rigidez
│   └── dor ao movimento
├── obstrutiva
│   ├── distensão
│   ├── vômitos
│   └── parada de gases
└── red flags
    ├── choque
    ├── sangramento
    └── peritonite
```

---

# 6. Critérios da ponte para OSCE

Toda trilha deve terminar mostrando como o conteúdo vira prática.

A ponte para OSCE deve responder:

```text
o que você pergunta?
o que você examina?
o que você fala em voz alta?
qual erro crítico deve evitar?
como apresenta o caso?
```

# 7. Fontes gerais de referência

As trilhas foram curadas com base em princípios consolidados de:

## Fisiologia e fisiopatologia

- Guyton & Hall — Tratado de Fisiologia Médica.
- Boron & Boulpaep — Medical Physiology.
- Ganong — Review of Medical Physiology.

## Semiologia

- Bates — Guide to Physical Examination and History Taking.
- Porto — Semiologia Médica.
- Macleod — Clinical Examination.

## Medicina interna

- Harrison — Principles of Internal Medicine.
- Cecil — Textbook of Medicine.
- Current Medical Diagnosis & Treatment.

## Urgências e segurança

- ACLS, PALS e ATLS conforme escopo de treinamento e supervisão.
- Protocolos institucionais locais.
- Diretrizes nacionais e internacionais de sociedades médicas quando aplicável.

## Consultas profissionais complementares

- MSD/Merck Manual Professional.
- BMJ Best Practice.
- UpToDate/DynaMed quando disponível institucionalmente.
- Artigos revisados por pares para temas específicos.

---

# 8. Critérios de atualização futura

Antes de uma trilha virar produto final comercial ou material docente oficial, deve passar por:

```text
1. revisão médica/docente;
2. checagem de diretrizes atualizadas;
3. validação de segurança;
4. teste visual no app;
5. teste de compreensão com estudante iniciante;
6. revisão de linguagem;
7. revisão de coerência com OSCE.
```

---

# 9. Limites do material

O MEDIX Learn V3:

- ensina raciocínio;
- não substitui professor;
- não substitui supervisão clínica;
- não define conduta real individual;
- não substitui protocolos locais;
- não deve ser usado como orientação médica para pacientes.

---

# 10. Checklist final de curadoria

Uma trilha está pronta quando:

- [ ] ensina mecanismo;
- [ ] conecta mecanismo a achado clínico;
- [ ] identifica gravidade;
- [ ] traz mini-casos específicos;
- [ ] tem questões ativas clínicas;
- [ ] tem mapa final útil;
- [ ] tem ponte para OSCE;
- [ ] evita conteúdo genérico;
- [ ] usa segunda pessoa;
- [ ] mantém segurança clínica.

## Frase de controle de qualidade

```text
Se o estudante terminar a trilha apenas decorando nomes, a trilha falhou.
Se ele terminar reconhecendo mecanismo, gravidade e decisão inicial, a trilha cumpriu seu papel.
```
